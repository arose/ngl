[name]

Selections (or 'Sele' for short) strings can be input at various places in the [page:User-interface user interface] or when scripting. They are used to limit which atoms/residues are shown in a [page:Molecular-representations molecular representation] or what atoms are loaded from a [page:Trajectory-access trajectory].

Language

Keywords

  • *sidechain*
  • *sidechainAttached*
  • *backbone*
  • *protein*
  • *nucleic*
  • *hetero*
  • *all*
  • *water*
  • *hydrogen*

Expressions

  • residue number: *1*, *2*, *100*
  • residue number range: *3-40* (Note that no spaces around - are allowed)
  • chain name: :A
  • atom name: .CA, .C, .N, ...
  • model: /0, /1, ...
  • residue name: *ALA*, *GLU*, *SOL*, *DMPC*, ...
  • element name: #H, #C, #O, ...
Some of these expressions can be combined (in this order) - residue numer (range), chain name, atom name, model - like this 10-20:A.CA/0 # select C-alpha atoms of residues 10 to 20 from chain A in model 0 which is the same as 10-20 and :A and .CA and /0 Single expressions may be left out ss long as the order (see above) is kept, e.g. :A/0 # select chain A from model 0

Logical operators (in order of binding strength)

  • *NOT*
  • *AND*
  • *OR*
Additionally, parentheses *()* can be used for grouping.

Examples

Select the side-chain and C-alpha atoms plus the backbone nitrogen in case of proline. not backbone or .CA or (PRO and .N) This selection is useful to display the sidechains together with a backbone trace. Hence there is also the keyword *sidechainAttached* for it.
